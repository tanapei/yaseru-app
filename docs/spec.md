# YASERU アプリ仕様書

## 1. 目的

「絶対に痩せたい」という強い意思を、毎日の迷いない行動に変えるためのスマホ向け管理アプリを作る。

このアプリは以下を最小操作で支援する。

- いつまでにどれくらい痩せるかを決める
- 食事、運動、睡眠を毎日記録する
- 今日やるべきことと達成状況を一目で把握する
- スクロールせず、1画面で必要情報を確認できる

## 2. プロダクトコンセプト

### コンセプト

「1日1画面、3管理、迷わない」

### 解決したい課題

- 目標体重や期限を決めても日々の行動に落ちない
- 記録アプリは入力項目が多く、続かない
- 食事、運動、睡眠が別々で管理され、全体感が見えない
- 毎日見る画面が複雑だと、開くこと自体が面倒になる

### 提供価値

- 開いた瞬間に「あと何kg」「あと何日」「今日何をやるか」がわかる
- 食事、運動、睡眠を各1〜2タップで記録できる
- 頑張りすぎではなく、継続しやすい行動管理に集中できる

## 3. ターゲットユーザー

- スマホ中心で使いたい人
- 記録が苦手で、入力が面倒だと続かない人
- まずは毎日続けることを最優先にしたい人
- 詳細分析より「今日やること」と「進捗の見える化」を求める人

## 4. 対応プラットフォーム

- スマホブラウザで利用できる Web アプリ
- 将来的にホーム画面追加しやすい PWA 構成を想定

## 5. UX の基本方針

### 最重要方針

- 縦スクロールなしで主要情報が見える
- 片手で操作しやすい
- 毎日の操作は最短 3 タップ以内を目指す
- 迷う自由入力を減らし、選択式を中心にする

### UI 原則

- 1画面完結のホームを最優先にする
- 入力はモーダルまたはボトムシートで行う
- 数字より先に状態を見せる
- 達成感が出る表現を使う

## 6. MVP の機能要件

MVP では以下に絞る。

### 6.1 初回目標設定

初回起動時に以下を設定する。

- 現在体重
- 目標体重
- 目標期限
- 1日の目標歩数または運動時間
- 1日の目標睡眠時間

設定完了後、ホーム画面に遷移する。

### 6.2 ホーム画面

ホーム画面はアプリの中心であり、毎日ここだけ見ればよい設計にする。

表示要素は以下。

- 目標サマリー
- 今日の進捗
- 食事カード
- 運動カード
- 睡眠カード
- 当日の総合達成率

### 6.3 食事管理

食事は厳密なカロリー計算ではなく、まず続けやすい自己管理を優先する。

記録内容

- 朝食: 食べた / 控えめ / 未記録
- 昼食: 食べた / 控えめ / 未記録
- 夕食: 食べた / 控えめ / 未記録
- 間食: なし / 少し / あり
- 水分: コップ数

入力方式

- ワンタップ選択
- 水分は `+1杯` ボタンで追加

MVP では栄養素詳細や食品検索は実装しない。

### 6.4 運動管理

記録内容

- 今日の運動実施: 実施 / 未実施
- 運動種別: 散歩 / 筋トレ / 有酸素 / ストレッチ / その他
- 運動時間: 5分刻みの選択式
- 歩数: 手入力

入力方式

- `運動した` ボタンで即記録
- 必要に応じて種別と時間を追加選択

### 6.5 睡眠管理

記録内容

- 睡眠時間
- 就寝時刻
- 起床時刻
- 睡眠の自己評価: 良い / 普通 / 悪い

入力方式

- 睡眠時間はプリセット選択
- 時刻は簡易入力
- 自己評価はワンタップ選択

### 6.6 体重進捗管理

記録内容

- 当日体重

表示内容

- 開始体重からの変化
- 目標体重まで残り何kgか
- 期限まで残り何日か
- 進捗率

体重入力はホーム上部からすぐ行えるようにする。

## 7. 画面仕様

### 7.1 画面一覧

MVP の画面は最小限の 3 画面構成とする。

- 初回設定画面
- ホーム画面
- 履歴画面

### 7.2 初回設定画面

目的

- 最初の 1 回で「いつまでにどれくらい痩せるか」を決める

要件

- 入力項目を 5 つ以内に抑える
- 1 画面または 2 ステップ以内で完了できる
- 完了ボタンを大きく配置する

表示項目

- 現在体重
- 目標体重
- 目標期限
- 運動目標
- 睡眠目標

### 7.3 ホーム画面

#### 目的

- 今日の管理を 1 画面で完了させる

#### ファーストビュー要件

スマホ縦画面でスクロールせず、以下がすべて見えること。

- 目標まで残り `◯kg`
- 期限まで残り `◯日`
- 今日の総合達成率
- 食事、運動、睡眠の現在ステータス
- 主要な記録ボタン

#### レイアウト案

1. 最上部: 目標サマリー
2. 中段: 今日の総合進捗
3. 下段: 食事、運動、睡眠の 3 カード
4. 最下部: クイック入力ボタン

#### 目標サマリー表示

- 現在体重
- 目標体重
- 残り kg
- 期限まで残り日数

#### 今日の総合進捗

- 円形または横バーで達成率を表示
- 食事、運動、睡眠の達成状況を集計

#### 3 カードの表示内容

食事カード

- 朝昼夕の状態
- 間食の有無
- 水分杯数
- `記録する` ボタン

運動カード

- 実施有無
- 運動時間
- 歩数
- `記録する` ボタン

睡眠カード

- 睡眠時間
- 就寝 / 起床
- 睡眠評価
- `記録する` ボタン

#### クイックアクション

- `体重入力`
- `食事入力`
- `運動入力`
- `睡眠入力`

### 7.4 履歴画面

目的

- 振り返り用に最低限の履歴を見る

表示内容

- 体重の推移
- 日別達成率
- 直近 7 日の記録一覧

MVP では詳細分析は行わない。

## 8. 主要操作フロー

### 8.1 初回利用

1. 現在体重を入力
2. 目標体重を入力
3. 目標期限を選択
4. 運動目標と睡眠目標を選択
5. ホームへ遷移

### 8.2 毎日の利用

1. ホームを開く
2. 体重を入力する
3. 食事、運動、睡眠の未記録項目だけ入力する
4. 今日の達成率を確認する

### 8.3 食事記録

1. `食事入力` をタップ
2. 朝昼夕、間食、水分を選択
3. 保存

### 8.4 運動記録

1. `運動入力` をタップ
2. 実施有無、種別、時間、歩数を入力
3. 保存

### 8.5 睡眠記録

1. `睡眠入力` をタップ
2. 睡眠時間、就寝 / 起床、睡眠評価を入力
3. 保存

## 9. 情報設計

### 9.1 優先度

ホーム上での情報優先度は以下。

1. 目標まで残り何kgか
2. 期限まで残り何日か
3. 今日の達成率
4. 未記録の項目
5. 体重、食事、運動、睡眠の詳細

### 9.2 色の考え方

- 達成: 前向きで見やすい色
- 未達: 強すぎない注意色
- 未記録: ニュートラル

赤系で罪悪感を強く煽る UI にはしない。

## 10. データ設計

### 10.1 ユーザー設定

- currentWeightStart
- targetWeight
- targetDate
- dailyExerciseGoalMinutes
- dailyStepGoal
- dailySleepGoalHours

### 10.2 日次記録

- date
- weight
- meals
- snackLevel
- waterCups
- exerciseDone
- exerciseType
- exerciseMinutes
- steps
- sleepHours
- bedtime
- wakeTime
- sleepQuality

### 10.3 達成率ロジック

MVP ではシンプルなスコアリングにする。

- 食事: 0〜100 点
- 運動: 0〜100 点
- 睡眠: 0〜100 点
- 総合達成率 = 3 項目の平均

食事スコア例

- 朝昼夕の記録完了
- 間食が少ない
- 水分目標を達成

運動スコア例

- 運動時間が目標以上
- または歩数目標を達成

睡眠スコア例

- 目標睡眠時間を満たす
- 睡眠評価が普通以上

※ 厳密な医学的指標ではなく、継続のための自己管理スコアとして扱う。

## 11. バリデーションと注意事項

- 目標体重は現在体重より小さい必要がある
- 目標期限は今日より未来日である必要がある
- 極端な目標設定時は注意メッセージを表示する
- アプリは医療判断を行わない

注意メッセージ例

- 「短期間で大きく減らす目標です。無理のない範囲で見直してください」

## 12. 非機能要件

- スマホで軽快に動作する
- 390px 幅前後でも崩れない
- ホーム画面はスクロール不要
- 主要ボタンは親指で押しやすいサイズにする
- 入力完了までの導線を短くする

## 13. MVP では実装しないもの

- カロリー自動計算
- 食品データベース検索
- スマートウォッチ連携
- SNS 共有
- AI による食事解析
- 通知の細かな自動最適化
- 医療的アドバイス

## 14. 今後の拡張候補

- リマインダー通知
- 週間レビュー
- 目標再設定ガイド
- 写真による食事記録
- ヘルスケア連携
- 習慣継続日数の表示

## 15. 実装方針の提案

空リポジトリからの着手であれば、以下の方針が適している。

- モバイルファーストの Web アプリ
- 1 画面中心の UI
- ローカル保存を使った MVP
- 後から認証やクラウド同期を追加しやすい構成

推奨する初期実装順

1. 初回設定画面
2. ホーム画面
3. 3 管理の入力モーダル
4. ローカル保存
5. 履歴画面

## 16. 完成イメージ

ユーザーがアプリを開くと、まず以下が見える。

- 「あと 4.2kg」
- 「期限まであと 73日」
- 「今日の達成率 67%」

その下に食事、運動、睡眠の 3 つのカードが並び、未入力のものだけをタップして埋めれば、その日の管理が終わる。

この体験を最優先に設計する。
